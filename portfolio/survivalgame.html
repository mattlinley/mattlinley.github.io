Project Goals

1. Develop a fully functioning 3D Game
2. A stateful system where many objects in the world can be interacted with.
3. To be able to save and load the world state.

Tools Used

1. Game engine - Unity
2. Code editor - Visual Studio
3. Language - C#
4. Version control - Git / Github
5. Save game data format - JSON
6. Modeling - Blender

Lessons Learned

I initially had some trouble using Github with my local repository. As this is a 3D project the file size is larger and any push to Github would fail due to both individual file sizes over 100mb and also total project size over 2gb. I solved this issue by using gitignore to ignore temporary files generated by Unity and Visual Studio. I also added all of my downloaded assets to the ignore list and listed these in the README so that the project can be completely recreated from the Github repository. This last step is also important because most assets downloaded from the Unity asset store should not be stored in a public repository under the terms and conditions of their use.

In this project I implemented a number of gameplay features before I started working on the save game system, and while this worked it was a messy way to do it and in future projects I will plan to implement the save game system at the first opportunity and incorporate saving along with new features. I also learned a lot about the best way to process and store data using JSON, for example the inability to serialize Queues and how to work around that.

Notable Game Features

- All trees can be chopped down for logs, and they will regrow back to their inital state after two game days.
- A crafting system.
- A construction system to build a custom wood shelter with floors, walls, doorways and roofs.
- A quest system using Scriptable objects to add and track quests.
- A quest tracker system so the status of any quest can be checked.
- An animated NPC who can be interacted with to complete a quest and obtain rewards, with voiceovers.
- A fishing system with minigame to catch different types of fish.
- A farming system to grow fruit from seeds. The fruit can be picked and eaten.
- A seasonal weather system.
- A full save and load system, including the state of any chopped trees, the state of any growing plants and all other items in the game.
- Inventory system where items can be dragged between the player inventory, a quick access bar and also storage chests.
- Inventory items can be dropped into the world with a 3D model, and then picked up again.
